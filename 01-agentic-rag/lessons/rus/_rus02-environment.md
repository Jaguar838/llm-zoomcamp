Для этого модуля вам понадобится только Python с Jupyter.

## Предварительные требования

Вам понадобится следующее:

- Python (версии 3.14 или новее)
- Аккаунт [OpenAI](https://openai.com/) (или провайдер, совместимый с OpenAI, такой как Groq, Gemini или Ollama)
- Базовое знакомство с Python и командной строкой

## Создание проекта

Мы начнем с нуля — клонирование не требуется. Вы создадите проект сами, шаг за шагом.

Сначала установите `uv`. Это менеджер пакетов Python, на который я перевел все свои проекты, потому что он быстрый и удобный. Как только я начал им пользоваться, мне больше не захотелось возвращаться к старому.

На Mac или Linux:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

На Windows:

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

(Вы также можете использовать `pip install uv`, если предпочитаете.)

Создайте пустую папку для проекта и инициализируйте ее:

```bash
mkdir llm-zoomcamp-code
cd llm-zoomcamp-code
uv init
```

Это создаст файл `pyproject.toml` и базовую структуру проекта.

Теперь добавьте зависимости, которые нам понадобятся:

```bash
uv add requests minsearch openai jupyter python-dotenv
```

Это установит:

- `requests` — для загрузки набора данных FAQ из интернета
- `minsearch` — простая поисковая система, работающая в оперативной памяти, для индексации и поиска текста
- `openai` — клиент API OpenAI для вызова LLM
- `jupyter` — среда ноутбуков, в которой мы будем писать и запускать код
- `python-dotenv` — для загрузки ключей API из файла `.env`

## Настройка ключей API

Нам нужен ключ API для связи с LLM. Если вы используете OpenAI, вам нужно будет сначала внести немного денег на счет. Минимум составляет $5 (по состоянию на июнь 2026 года). Выполнение этого урока стоит меньше 10 центов, так что этих 5 долларов хватит надолго.

Я также рекомендую создать отдельный проект OpenAI для курса. Тогда вы сможете открыть страницу использования и увидеть, сколько именно вы потратили здесь, отдельно от вашей остальной работы.

Самый безопасный способ хранения ключа — в файле `.env`, который никогда не фиксируется (commit) в git.

Создайте файл `.env` в папке вашего проекта и поместите в него свой ключ API:

```bash
OPENAI_API_KEY=sk-YOUR_KEY_HERE
```

Теперь добавьте `.env` в `.gitignore`, чтобы случайно не закоммитить свой ключ:

```bash
.env
```

Никогда не коммитьте `.env` в git. Относитесь к ключу API как к паролю. Если он утечет, кто-то другой сможет совершать платные запросы за ваш счет.

## Запуск Jupyter

Запустите Jupyter:

```bash
uv run jupyter notebook
```

Создайте новый ноутбук. На протяжении всего курса вы будете копировать код из заметок к разделам в ячейки ноутбука.

Проверьте, что клиент OpenAI работает:

```python
from dotenv import load_dotenv
load_dotenv()

from openai import OpenAI
openai_client = OpenAI()
```

Если вы видите ошибку, убедитесь, что ключ в вашем файле `.env` указан правильно.

Для Groq или других провайдеров, совместимых с OpenAI, добавьте ключ в `.env`:

```bash
GROQ_API_KEY=your_key_here
```

И настройте клиент:

```python
from openai import OpenAI
import os

openai_client = OpenAI(
    api_key=os.getenv("GROQ_API_KEY"),
    base_url="https://api.groq.com/openai/v1"
)
```

## (Опционально) Автозагрузка .env с помощью dirdotenv

Если вы не хотите вызывать `load_dotenv()` в каждом ноутбуке, используйте [dirdotenv](https://github.com/alexeygrigorev/dirdotenv).

Он автоматически загружает файлы `.env`, когда вы переходите (`cd`) в директорию:

```bash
uv tool install dirdotenv
echo 'eval "$(dirdotenv hook bash)"' >> ~/.bashrc
```

Перезапустите терминал, и теперь всякий раз, когда вы входите в директорию проекта, переменные из `.env` будут загружаться автоматически. Вызов `load_dotenv()` не потребуется.
