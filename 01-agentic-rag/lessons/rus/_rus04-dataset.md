# Набор данных FAQ курса

Видео: [Смотреть этот урок](https://www.youtube.com/watch?v=Mx6EqvzVDz0&list=PL3MmuxUbc_hLZFNgSad56pDBKK8KO0XIv)

Прежде чем мы построим конвейер RAG, давайте познакомимся с данными, которые мы будем использовать в качестве нашей базы знаний.

Мы проводим эти курсы каждый год, и студенты постоянно задают одни и те же вопросы в Slack. Мы собрали их в FAQ (часто задаваемые вопросы), чтобы люди могли найти ответы самостоятельно. Некоторые курсы прошли уже пять потоков, поэтому FAQ разросся, и искать в нем ответы вручную стало утомительно. Именно эту проблему решит наша система RAG.

Данные FAQ доступны в формате JSON на сайте DataTalks.Club. Я поддерживаю этот сайт, поэтому сделал данные доступными через JSON-эндпоинт, который мы можем загрузить напрямую.

Давайте загрузим их:

```python
import requests

docs_url = "https://datatalks.club/faq/json/courses.json"
response = requests.get(docs_url)
courses_raw = response.json()
```

Это возвращает список курсов. У каждого курса есть поле `path`, которое указывает на его данные FAQ.

Загрузим все документы FAQ для всех курсов:

```python
documents = []
url_prefix = "https://datatalks.club/faq"

for course in courses_raw:
    course_url = f"""{url_prefix}{course["path"]}"""

    course_response = requests.get(course_url)
    course_response.raise_for_status()
    course_data = course_response.json()

    documents.extend(course_data)

len(documents)
```

Каждая запись содержит:

- `id` — уникальный идентификатор
- `course` — идентификатор (slug) курса (например, `machine-learning-zoomcamp`)
- `section` — раздел курса
- `question` — вопрос из FAQ
- `answer` — ответ из FAQ

Давайте посмотрим на одну запись:

```python
documents[0]
```

Вы должны увидеть что-то вроде:

```python
{
    "id": "0e38656cfb",
    "course": "machine-learning-zoomcamp",
    "section": "General Course-Related Questions",
    "question": "How do I submit homework?",
    "answer": "- Do the tasks locally\n- Publish your code ..."
}
```

У каждого курса есть slug — короткий идентификатор, используемый в URL. Например, `machine-learning-zoomcamp`, `data-engineering-zoomcamp` и т. д. Мы будем использовать эти идентификаторы для фильтрации при поиске.

## Использование этих данных

В конвейере RAG этот набор данных является нашей базой знаний:

1. Мы индексируем все документы (шаг поиска)
2. Когда студент задает вопрос, мы ищем его в индексе
3. Поиск возвращает наиболее подходящие записи FAQ
4. Мы передаем эти записи LLM в качестве контекста
5. LLM генерирует ответ на основе контекста

Поля `question` и `answer` содержат текст, по которому мы будем искать. Поле `course` позволяет нам фильтровать результаты по конкретному курсу. Например, если студент спрашивает о курсе по инженерии данных, мы пропускаем результаты из курса по машинному обучению. Поле `section` помогает в ранжировании — знание того, к какой части курса относится вопрос, является полезным контекстом.

## Примечание по подготовке данных

 В нашем случае данные уже подготовлены. Я поддерживаю этот сайт FAQ и позаботился о том, чтобы данные возвращались в удобном формате JSON. Поэтому нам не нужно много делать, чтобы привести их в порядок. Кстати, я почистил значительную часть этих данных с помощью LLM. Это само по себе полезный сценарий использования.

Пусть это не вводит вас в заблуждение. В реальности подготовка данных часто является самой трудоемкой частью создания системы RAG. Вам может потребоваться парсить сайты, обрабатывать PDF-файлы, а также чистить и разбивать документы на части (chunking). Эта работа здесь не видна, но я проделал ее заранее.
