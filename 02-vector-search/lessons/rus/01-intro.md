# Векторный поиск

Видео: [Посмотреть этот урок](https://www.youtube.com/watch?v=qyZgxTmC2cY&list=PL3MmuxUbc_hLZFNgSad56pDBKK8KO0XIv)

В модуле 1 мы использовали поиск по ключевым словам с помощью minsearch и sqlitesearch.
Он ищет точные совпадения слов. Если вы ищете "Docker", документ должен
содержать слово "Docker", чтобы попасть в результаты.

Но посмотрите на эти два вопроса:

- "Can I still join the course after the start date?" (Могу ли я присоединиться к курсу после даты начала?)
- "Is it possible to enroll late?" (Возможна ли поздняя регистрация?)

Они означают одно и то же, хотя в них почти нет общих слов. Поисковой движок по ключевым
словам с трудом сопоставит их. Нам нужно что-то, что работает со смыслом,
а не с точными словами.

Это "что-то" — векторный поиск. Вместо сопоставления слов он сопоставляет идеи.

## Процесс векторного поиска

Мы запускаем векторный поиск в два этапа.

1. Оффлайн (индексация): мы преобразуем все документы в векторы (массивы
   чисел) и сохраняем их в индексе.
2. Онлайн (запросы): мы преобразуем запрос пользователя в вектор с помощью
   той же модели, а затем находим наиболее близкие векторы документов по сходству.

Модель эмбеддингов (embedding model) создает эти векторы. Это нейронная сеть,
обученная улавливать смысл, поэтому тексты со схожим значением превращаются
в похожие векторы. Мы измеряем близость двух векторов с помощью метрики расстояния.
 Самая распространенная — косинусное сходство (cosine similarity).

Косинусное сходство измеряет угол между двумя векторами:

- Векторы, направленные в одну сторону: сходство близко к 1 (похожи)
- Векторы под прямым углом: сходство близко к 0 (не связаны)
- Векторы, направленные в противоположные стороны: сходство близко к -1 (противоположный смысл)

Чем больше косинусное сходство, тем более похожи два текста по смыслу.

## Поиск по ключевым словам против векторного поиска

Вот чем различаются эти два подхода:

- Поиск по ключевым словам ищет точные слова. Векторный поиск ищет смысл.
- Поиск по ключевым словам подходит для специфических терминов, ID и имен. Векторный
  поиск подходит для перефразированных вопросов и естественного языка.
- Пример поиска по ключевым словам: "pandas dataframe". Пример векторного
  поиска: "How do I work with tabular data?" (Как мне работать с табличными данными?)
- Поиск по ключевым словам использует инвертированный индекс (BM25, TF-IDF).
  Векторный поиск использует векторный индекс на основе косинусного сходства.
- Поиск по ключевым словам упускает синонимы и парафразы. Векторный поиск упускает
  точные совпадения терминов.

Векторный поиск обычно лучше, но он добавляет много операционной сложности,
и вы почувствуете это на протяжении всего модуля. Поэтому мой совет — никогда не
начинайте с векторного поиска. Начните с текстового поиска и переходите к векторам
только тогда, когда сможете доказать, что они стоят дополнительных затрат.

На практике эти два метода лучше всего работают вместе. Гибридный поиск объединяет их,
и мы рассмотрим его в [модуле по лучшим практикам](../../06-best-practices/lessons/02-hybrid-search.md).

## Создание векторного поиска

Мы возьмем тот же набор данных FAQ из модуля 1 и построим векторный поиск
с помощью трех инструментов:

1. minsearch — векторный поиск в оперативной памяти (самый простой, хорош для экспериментов)
2. sqlitesearch — постоянный векторный поиск на базе SQLite (подходит для продакшена,
   тот же API, что и у minsearch)
3. PGVector — векторный поиск в PostgreSQL (масштабируемый, запускается в Docker)

Затем мы подключим векторный поиск в наш RAG-конвейер.

Код из этого модуля доступен в [директории с кодом](../code/).

установим sentence-transformers. Она подтягивает PyTorch и довольно тяжелая,

```bash
cd llm-zoomcamp-code
uv init
uv add requests minsearch openai jupyter python-dotenv
```

Вам также понадобится файл `.env` с вашим API-ключом. Подробности см. в
[настройке окружения модуля 1](../../01-agentic-rag/lessons/02-environment.md).
