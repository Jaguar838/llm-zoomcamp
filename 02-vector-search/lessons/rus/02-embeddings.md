# Эмбеддинги

Видео: [Посмотреть этот урок](https://www.youtube.com/watch?v=kJOlW1HeMp4&list=PL3MmuxUbc_hLZFNgSad56pDBKK8KO0XIv)

Прежде чем мы сможем выполнять векторный поиск, нам нужно превратить наш текст в векторы.
Мы называем этот процесс получением эмбеддингов (embedding): мы встраиваем текст в векторное
пространство. Полученные векторы также называют "эмбеддингами".

## Эмбеддинги слов и предложений

Эта идея пришла из [word2vec](https://en.wikipedia.org/wiki/Word2vec). Модель учится
размещать слова как точки в многомерном пространстве. Слова с похожим значением
оказываются близко друг к другу.

Представьте 2D-пространство, где "enroll" (записаться) и "join" (присоединиться) находятся
рядом, а "Docker" далеко:

```text
        · enroll
       · join
                   · Docker
```

Та же идея работает для целых предложений:

```text
Q1: "I just discovered the course. Can I still join it?"
Q2: "I just found out about the program. Can I still enroll?"

Эти два предложения близки — они означают одно и то же.

Q3: "How do I run Docker on Windows?"

Это предложение находится далеко от Q1 и Q2.
```

Теперь представьте все 1200 документов в нашем наборе данных FAQ. Каждый из них становится
точкой в этом пространстве. Когда пользователь задает вопрос, мы переводим его в то же
пространство и находим ближайшие документы. Эти "ближайшие соседи" и являются нашими
результатами поиска.

Модель кодирует все предложение целиком, а не слова по отдельности. Поэтому она может
различать одно и то же слово в разных контекстах.

Возьмем слово "judge". В предложении "the judge ruled out the possibility of crime"
(судья исключил возможность преступления — юридический контекст) оно получит один вектор.
В "LLM-as-a-judge approach to evaluate LLMs" (подход LLM-как-судья для оценки LLM —
контекст машинного обучения) оно получит другой вектор. Окружающий контекст меняет эмбеддинг.

Таким образом, модель эмбеддингов принимает текст на вход и возвращает массив чисел
фиксированной длины. Мы обучаем её так, чтобы тексты со схожим смыслом получали похожие векторы.

Мы будем использовать [sentence-transformers](https://www.sbert.net/), популярную
библиотеку с открытым исходным кодом для создания эмбеддингов. Она работает локально
на вашем компьютере, поэтому затраты на API отсутствуют.

## Установка sentence-transformers

Установите библиотеку:

```bash
uv add sentence-transformers
```

Это также подтянет PyTorch, так что скачиваться будет много. Вы увидите, как пролетают
пакеты CUDA и другие пакеты Nvidia. Это нормально для экспериментов, и мы сократим
их размер для продакшена в [одном из следующих уроков](09-onnx-embedder.md).

## Выбор модели

Sentence-transformers поддерживает множество моделей. Правильный выбор зависит от вашей
задачи, языка и имеющихся ресурсов. Более крупные модели обычно медленнее, поэтому для
нашего набора данных FAQ с короткими английскими текстами достаточно небольшой модели.
Попробуйте несколько на своих данных и оставьте ту, которая работает лучше всего.

Мы будем использовать `all-MiniLM-L6-v2`:

- 384-мерные векторы (компактные)
- Быстрая работа на CPU
- Хорошее качество для общего английского текста
- Использует косинусное сходство (мы объясним это ниже)

```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("all-MiniLM-L6-v2")
```

При первом запуске скачивается модель (~80 МБ) и токенизатор с HuggingFace. Токенизатор
превращает текст в нечто, что модель может прочитать. После этого оба загружаются из
локального кэша.

## Пробуем на простых примерах

Давайте посмотрим, как работают эмбеддинги на нескольких примерах.

Начнем с запроса:

```python
q1 = "Can I still join the course after the start date?"
v1 = model.encode(q1)
```

`v1` — это вектор, массив из 384 чисел. Каждое число соответствует какой-то концепции,
которую выучила модель. Мы не можем напрямую прочитать, что означает каждое из них.
Но два вектора со схожими значениями указывают на тексты о похожих вещах.

Закодируем наш документ:

```python
d  = "You don't need to register. You're accepted. You can also just start learning and submitting homework without registering."
dv = model.encode(d)
```

Затем мы сравниваем запрос с документом, используя скалярное произведение (dot product):

```python
v1.dot(dv)
```

Мы получаем 0.32.

Теперь попробуем несвязанный запрос:

```python
q2 = "How to install Docker on Windows?"
v2 = model.encode(q2)
```

На этот раз сходство с документом должно быть намного меньше:

```python
v2.dot(dv)
```

И мы получаем 0.01.

Первый результат для `q1` против `d` (0.32) выше, так что этот запрос более похож на
документ о регистрации. Второй результат для `q2` против `d` близок к 0, потому что
установка Docker не имеет ничего общего с регистрацией. Оценка, близкая к 0, означает,
что два вектора максимально различны.

В этом и заключается вся идея векторного поиска: похожие тексты получают похожие векторы,
а скалярное произведение говорит нам, насколько они похожи.

## Косинусное сходство

Модель `all-MiniLM-L6-v2` выдает нормализованные векторы — векторы единичной длины.
Когда оба вектора нормализованы, скалярное произведение равно косинусному сходству.
Вот почему в документации модели сказано, что она "использует косинусное сходство".

Косинусное сходство измеряет угол между двумя векторами, игнорируя их длину:

- 1.0 = то же направление (похожи)
- 0.0 = перпендикулярно (не связаны)
- -1.0 = противоположное направление (противоположный смысл)

Формально, если `theta` — угол между двумя векторами, косинусное сходство — это `cos(theta)`:

- `cos(0) = 1` — векторы направлены в одну сторону
- `cos(90) = 0` — векторы перпендикулярны
- `cos(180) = -1` — векторы направлены в противоположные стороны

Поскольку наши векторы нормализованы, скалярное произведение дает нам косинусное сходство
напрямую. Вот почему мы можем использовать `v1.dot(dv)` для сравнения текстов.

На практике мы редко получаем косинусное сходство ниже 0. Модель эмбеддингов отображает
текст в область векторного пространства, где большинство векторов имеют положительные
компоненты. Концепции "противоположного смысла", которая отображалась бы в вектор,
направленный в другую сторону, не существует.
