# Векторный поиск

Видео: [Посмотреть этот урок](https://www.youtube.com/watch?v=h-_tdBc24qc&list=PL3MmuxUbc_hLZFNgSad56pDBKK8KO0XIv)

В предыдущем уроке мы встроили наш набор данных FAQ в матрицу `X`, содержащую 1208
векторов документов. Здесь мы увидим, как векторный поиск работает "под капотом".

## Оценка документов

У нас есть матрица `X` со всеми эмбеддингами документов. Мы берем запрос, сравниваем
его с каждым документом и оставляем наиболее похожие.

Когда поступает запрос, мы превращаем его в эмбеддинг:

```python
query = "Can I still join the course after the start date?"
v_query = model.encode(query)
```

Затем мы вычисляем скалярное произведение (dot product) со всеми документами:

```python
scores = X.dot(v_query)
```

Это умножение матрицы на вектор. Каждый элемент `i` массива `scores` — это косинусное
сходство между документом `i` (строка `i` матрицы `X`) и `v_query`.

Мы могли бы вычислить то же самое с помощью цикла for:

```python
scores = [v_query.dot(X[i]) for i in range(len(X))]
```

Но `X.dot(v_query)` работает намного быстрее. Numpy запускает оптимизированный код на C
вместо цикла на Python, поэтому матричную версию трудно превзойти. Результат в любом
случае одинаков: одна оценка на каждый документ.

## Лучшее совпадение

Наивысший балл соответствует самому похожему документу:

```python
idx = np.argmax(scores)
idx, scores[idx]
```

Это вернет документ 553 с оценкой 0.76.

Индекс и оценка у вас могут отличаться. Наш FAQ — это живой документ, поэтому мы со
временем добавляем и удаляем записи.

Давайте посмотрим, что это за документ:

```python
documents[idx]
```

Мы увидим:

```python
{"id": "3f1424af17",
 "course": "data-engineering-zoomcamp",
 "section": "General Course-Related Questions",
 "question": "Course: Can I still join the course after the start date?",
 "answer": "Yes, even if you don't register, you're still eligible..."}
```

## Топ-5 результатов

Обычно нам нужно больше, чем одно лучшее совпадение, поэтому давайте выберем топ-5.

`np.argsort` сортирует от меньшего к большему, поэтому последние 5 элементов — это лучшие:

```python
top5 = np.argsort(scores)[-5:]
```

Они выводятся в порядке возрастания (от меньшего к большему), поэтому мы переворачиваем
их, чтобы получить лучшие первыми:

```python
top5 = top5[::-1]
top5
```

Теперь мы можем прочитать топ-5 оценок:

```python
scores[top5]
```

Есть более короткий трюк, который я обычно использую. Сначала мы меняем знак оценок на
противоположный, чтобы самое большое значение стало самым маленьким. Тогда `argsort`
поместит лучшие совпадения в начало.

Вот как это выглядит в одну строку:

```python
top5 = np.argsort(-scores)[:5]
```

В первый раз это может выглядеть загадочно. Но это обычный способ превратить сортировку
по минимуму в сортировку по максимуму.

Давайте выведем сами документы:

```python
for idx in top5:
    print(scores[idx])
    print(documents[idx])
    print()
```

Это векторный поиск в его простейшем виде. Мы превращаем запрос в эмбеддинг, вычисляем
скалярные произведения со всеми документами и возвращаем те, у которых самые высокие оценки.

Мы возвращаем 5 документов, а не один лучший, по определенной причине. Ответ на вопрос
может быть распределен по нескольким документам. Один содержит часть ответа, другой дополняет
его. Иногда лучший результат оказывается не совсем верным, а второй — правильным. Мы
отправляем все 5 документов в LLM и позволяем ей объединить их.

Число 5 — это отправная точка, выбранная по интуиции. Позже, когда мы будем оценивать
качество поиска, мы сможем проверить, не работают ли 3 или 10 документов лучше для наших данных.

Делать это вручную с помощью numpy удобно для небольшого набора данных. Для более крупного
понадобится библиотека, которая также умеет фильтровать и ранжировать. К этому мы и
перейдем далее.
