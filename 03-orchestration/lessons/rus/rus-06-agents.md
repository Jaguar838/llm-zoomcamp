# ИИ-агенты

Видео: [Агентные рабочие процессы](https://www.youtube.com/watch?v=7tvpR8EE0gs&list=PL3MmuxUbc_hLZFNgSad56pDBKK8KO0XIv)

В [Модуле 1](../../01-agentic-rag/lessons/11-agents-intro.md) вы вручную создавали агентный цикл: цикл `while`, который вызывал LLM, выполнял возвращаемые вызовы инструментов, отправлял результаты обратно и останавливался, когда модель выдавала окончательный ответ без дополнительных вызовов инструментов. Этот паттерн является основой любого агентного фреймворка.

В Kestra плагин `AIAgent` берет этот цикл на себя. Вы определяете цель, инструменты и, опционально, системное сообщение — Kestra управляет циклом, историей диалога и выдает результат как выходные данные задачи.

> Примечание: Потоки в этом уроке используют `{{ secret('GEMINI_API_KEY') }}`. Перед их запуском убедитесь, что вы выполнили [инструкции по настройке](rus-03-setup.md) для конфигурации этого секрета.

Примеры потоков используют Gemini, но блок `provider` поддерживает любого крупного провайдера ИИ — замените `io.kestra.plugin.ai.provider.GoogleGemini` на `OpenAI`, `Anthropic` или других. См. [полный список поддерживаемых провайдеров](https://kestra.io/plugins/plugin-ai/provider).

Традиционный рабочий процесс — фиксированная последовательность, заранее определенная логика:

```yaml
tasks:
  - id: step1
    type: Task1
  - id: step2
    type: Task2
  - id: step3
    type: Task3
```

Рабочий процесс с ИИ-агентом — агент решает, что делать и в каком порядке, исходя из цели:

```yaml
tasks:
  - id: agent
    type: io.kestra.plugin.ai.agent.AIAgent
    prompt: "Исследуй тренды в инженерии данных и создай отчет"
    tools:
      - WebSearch
      - TaskExecution
```

## Когда использовать ИИ-агентов

Используйте ИИ-агентов, когда точная последовательность шагов не известна заранее, решения зависят от динамически меняющейся информации или вам нужно адаптироваться к неожиданным условиям.

Используйте традиционные рабочие процессы, когда шаги детерминированы и повторяемы, соответствие нормативным требованиям требует точных аудируемых процессов, или необходимо минимизировать затраты и задержки.

## Анатомия ИИ-агента

```yaml
id: example_agent
namespace: zoomcamp

tasks:
  - id: agent
    type: io.kestra.plugin.ai.agent.AIAgent

    # Определяет роль и поведение агента
    systemMessage: |
      Вы — аналитик данных. Анализируйте данные и предоставляйте инсайты.

    # Сама задача или вопрос
    prompt: "Каковы 3 основных тренда в этих данных?"

    # Конфигурация провайдера LLM
    provider:
      type: io.kestra.plugin.ai.provider.GoogleGemini
      modelName: gemini-2.5-flash
      apiKey: "{{ secret('GEMINI_API_KEY') }}"

    # Инструменты, которые может использовать агент
    tools:
      - type: io.kestra.plugin.ai.tool.TavilyWebSearch
        apiKey: "{{ secret('TAVILY_API_KEY') }}"

    # Память для сохранения контекста между запусками
    memory:
      type: io.kestra.plugin.ai.memory.KestraKVStore
      memoryId: analyst_001
```

## Пример продвинутого агента: Веб-исследование

Поток: [`5_web_research_agent.yaml`](../flows/5_web_research_agent.yaml)

Этот поток демонстрирует агента с автономным использованием инструментов:

1. Получает промпт для исследования (например, "Последние тренды в оркестрации рабочих процессов")
2. Решает использовать инструмент веб-поиска для сбора информации
3. Оценивает результаты поиска и определяет, нужны ли дополнительные запросы
4. Синтезирует результаты в структурированный отчет в формате markdown
5. Сохраняет отчет в файл с помощью инструмента файловой системы

Агент автономно решает, когда использовать инструменты, может зацикливаться (поиск → оценка → повторный поиск) до достижения результата, а вы указываете только цель — а не точные шаги.

## Инструменты агентов, доступные в Kestra

| Инструмент | Цель | Пример использования |
|------------|------|----------------------|
| `TavilyWebSearch` | Поиск в вебе актуальной информации | Исследование рынка, мониторинг новостей |
| `GoogleCustomWebSearch` | Поиск через Google Custom Search API | Поиск в Google |
| `CodeExecution` | Безопасный запуск кода через Judge0 | Математические вычисления, валидация данных |
| `KestraTask` | Выполнение любой задачи Kestra | Запуск задач на базе 1000+ плагинов Kestra |
| `KestraFlow` | Запуск других потоков Kestra | Вызов других потоков для модульности |
| `StreamableHttpMcpClient` | Использование MCP-серверов через HTTP/SSE | Подключение к удаленным MCP-серверам |
| `DockerMcpClient` | Использование MCP-серверов в Docker | MCP-серверы, запускаемые по требованию через Docker |
| `StdioMcpClient` | Использование MCP-серверов через stdio | Интеграция с внешними системами |
| `AIAgent` | Использование другого агента как инструмента | Мультиагентные системы, специализированные субагенты |

## Наблюдаемость агентов

Kestra обеспечивает полную наблюдаемость выполнения агентов — использование токенов, выполнение инструментов, логи запросов и ответов, результаты и время выполнения.

Включите детальное логирование через свойство `configuration`:

```yaml
tasks:
  - id: research_agent
    type: io.kestra.plugin.ai.agent.AIAgent
    description: Автономный агент для исследований с возможностью веб-поиска
    provider:
      type: io.kestra.plugin.ai.provider.GoogleGemini
      apiKey: "{{ secret('GEMINI_API_KEY') }}"
      modelName: gemini-2.5-flash
    configuration:
      logRequests: true
      logResponses: true
```
