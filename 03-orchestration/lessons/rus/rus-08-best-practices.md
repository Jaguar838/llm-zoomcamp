# Лучшие практики

## Что и когда использовать

| Сценарий | Что использовать | Почему |
|----------|----------|-----|
| Создание/редактирование потоков | AI Copilot | Самый быстрый способ генерации YAML-кода потоков |
| Ответы на вопросы о ваших данных | RAG | Обосновывает ответы на реальных данных |
| Фиксированные, повторяемые ETL-пайплайны | Традиционные рабочие процессы | Детерминированные, предсказуемые, соответствующие стандартам |
| Задачи исследования и анализа | ИИ-агенты | Могут адаптироваться к полученным результатам и принимать решения |
| Сложные многоэтапные цели | Мультиагентные системы | Специализированные агенты, работающие вместе |

## Рекомендации по затратам

Функции ИИ используют API больших языковых моделей (LLM), стоимость которых зависит от количества использованных токенов.

Цены за 1 млн токенов ([полная страница с ценами](https://ai.google.dev/gemini-api/docs/pricing)):

| Модель | Уровень | Входные токены | Выходные токены |
|-------|------|-------|--------|
| Gemini 2.5 Flash | Free | $0.00 | $0.00 |
| Gemini 2.5 Flash | Batch / Flex | $0.15 | $1.25 |
| Gemini 3.5 Flash | Free | $0.00 | $0.00 |
| Gemini 3.5 Flash | Standard | $1.50 | $9.00 |
| Gemini 3.5 Flash | Batch / Flex | $0.75 | $4.50 |
| Gemini 3.5 Flash | Priority | $2.70 | $16.20 |

Для большинства рабочих процессов используйте Gemini 2.5 Flash — она дешевле и бесплатна для стандартных запросов. Переходите на Gemini 3.5 Flash, когда вам требуются более сильные способности к рассуждению для сложных агентных задач.

Советы по экономии:

1. Начинайте с бесплатного уровня для обучения и разработки.
2. Используйте модели меньшего размера/более дешевые для простых задач — проверяйте [страницу с ценами](https://ai.google.dev/gemini-api/docs/pricing).
3. Устанавливайте `maxOutputTokens`, чтобы ограничить размер ответа.
4. Отслеживайте использование токенов в результатах выполнения.
5. Используйте традиционные рабочие процессы там, где важна детерминированность.

## Безопасность

Никогда не фиксируйте API-ключи в Git. Всегда используйте секреты:

```yaml
# ❌ Неправильно
apiKey: "sk-abc123def456"

# ✅ Правильно
apiKey: "{{ secret('GEMINI_API_KEY') }}"
```

Экспортируйте ключи, закодированные в base64, как переменные окружения с префиксом `SECRET_` перед запуском Kestra. Регулярно меняйте ключи (например, каждые 90 дней) и следите за их использованием. Подробнее о секретах читайте в [документации Kestra](https://kestra.io/docs/concepts/secret).

## Наблюдаемость и отладка

При поиске неисправностей включайте детальное логирование:

```yaml
- id: my_agent_task
  type: io.kestra.plugin.ai.agent.AIAgent
  provider:
    # ...
    # настройки провайдера
    # ...
  configuration:
    logRequests: true
    logResponses: true
```

Мониторьте использование токенов для каждого запуска, вызовы инструментов и решения агентов, время выполнения, затраты и качество результата.

Советы по отладке:

1. Начинайте с простых промптов и итерируйтесь.
2. Проверяйте логи на предмет логики рассуждений LLM.
3. Проверяйте выходные данные выполнения инструментов.

## Готовность к эксплуатации (Production)

Перед развертыванием рабочих процессов с ИИ в продакшене:

1. **Тщательно тестируйте** — запускайте многократно с разными входными данными, проверяйте согласованность и точность результатов.
2. **Добавляйте запасные варианты (fallbacks)** — обрабатывайте сбои API с помощью повторных попыток и настраивайте оповещения об ошибках.
3. **Устанавливайте лимиты** — ограничивайте `maxOutputTokens` для контроля затрат.
4. **Документируйте поведение** — объясняйте, что делает агент, в описаниях ваших потоков и задач.
