# Чат-приложение

Видео: [Посмотреть этот урок](https://www.youtube.com/watch?v=JCB4JZlMsIQ&list=PL3MmuxUbc_hLZFNgSad56pDBKK8KO0XIv)

Командная строка работает, но нам нужно что-то более близкое к тому, как человек на самом деле будет общаться с ассистентом. Поэтому мы обернем его в небольшой веб-интерфейс. Мы будем использовать Streamlit — фреймворк на Python для создания фронтенда практически без написания кода.

Это не окончательный продукт, и он не обязан быть красивым. Я намеренно сделал его простым, чтобы не пришлось ничего объяснять. Если вам нужен более приятный интерфейс, обратитесь к ИИ-помощнику, например Claude Code или Codex, и попросите его улучшить верстку. Пока что простого варианта достаточно.

Добавьте Streamlit в свой проект:

```bash
uv add streamlit
```

Создайте `app.py`:

```python
import streamlit as st
from assistant import create_assistant

assistant = create_assistant()

st.title("Помощник по курсу")

user_input = st.text_input("Введите ваш вопрос:")

if st.button("Спросить"):
    with st.spinner("Обработка..."):
        answer = assistant.rag(user_input)
        st.success("Готово!")
        st.write(answer)
```

Запустите приложение:

```bash
uv run streamlit run app.py
```

Это еще одна длинная команда, поэтому мы также добавим её в `Makefile`:

```makefile
chat:
	uv run streamlit run app.py
```

Теперь мы можем запустить:

```bash
make chat
```

Если вы работаете в GitHub Codespaces, он сам обнаружит порт и пробросит его для вас. Откройте ссылку, и вы увидите веб-интерфейс. Задайте вопрос, например "How do I join the course?", и вы получите ответ.

RAG работает, но на данный момент мы ничего не отслеживаем: ни время ответа, ни использование токенов, ни стоимость. Именно такую прозрачность и должен обеспечивать мониторинг, и это то, что мы добавим далее.
