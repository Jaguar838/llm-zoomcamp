# Дашборды Grafana

Видео: [Посмотреть этот урок](https://www.youtube.com/watch?v=Pmh2jT8tEiw&list=PL3MmuxUbc_hLZFNgSad56pDBKK8KO0XIv)

Наш дашборд Streamlit уже работает. Мы подключаем Grafana, потому что она умеет гораздо больше. Это специализированный инструмент для создания дашбордов: она подключается ко многим источникам данных и предлагает больше типов панелей. Она также может отправлять уведомления (алерты), когда метрика пересекает определенный порог.

Минус в том, что это отдельное, более тяжелое приложение. Именно поэтому, если ваши потребности скромны, дашборда на Streamlit будет достаточно. Когда же вам нужно больше возможностей, Grafana считывает данные напрямую из уже имеющейся у нас базы Postgres и показывает происходящее в реальном времени.

## Запуск Grafana с помощью Docker

Запустите Grafana в той же сети с томом для сохранения данных:

```bash
docker run -d \
    --name grafana \
    --network monitoring \
    -p 3000:3000 \
    -v grafana_data:/var/lib/grafana \
    grafana/grafana
```

Мы уже создали сеть и запустили в ней PostgreSQL в уроке по базам данных. Мы запускаем Grafana в фоновом режиме с флагом `-d`. Если вы предпочитаете видеть логи во время настройки, уберите `-d`. Благодаря тому (volume) всё, что вы создадите в Grafana (источники данных и дашборды), сохранится после перезапуска.

Grafana доступна по адресу `http://localhost:3000`. Первый вход: логин `admin`, пароль `admin`. Система попросит вас установить новый пароль. Для локальной работы можно снова поставить `admin`.

## Настройка источника данных

Подключите Grafana к PostgreSQL:

1. Перейдите в Configuration > Data Sources > Add data source
2. Выберите PostgreSQL
3. Заполните данные для подключения:
   - Host: `course-assistant-pg:5432`
   - Database: `course_assistant`
   - User: `user`
   - Password: `password`
   - SSL Mode: disable
4. Нажмите Save & Test. Должно появиться сообщение "Database Connection OK"

## Создание дашборда

Создайте новый дашборд. Мы будем добавлять панели по одной. Каждая панель — это SQL-запрос, который Grafana выполняет к PostgreSQL.

Чтобы запросы работали корректно, следуйте двум правилам. Во-первых, давайте колонке с временем псевдоним `time`. Grafana использует её для оси X на графиках. Во-вторых, фильтруйте данные по выбранному временному диапазону. Тогда панель будет показывать данные только за тот период, который вы выбрали в верхней части экрана.

Для такой фильтрации в Grafana есть специальные переменные:

- `$__timeFrom()`: начало выбранного диапазона
- `$__timeTo()`: конец выбранного диапазона
- `$__timeGroup(column, interval)`: группировка результатов по временным интервалам

Использование `$__timeFrom()` и `$__timeTo()` в блоке `WHERE` не всегда обязательно, но лучше указывать их явно. Так панель будет четко следовать выбранному диапазону.

## Панель времени ответа (Response Time)

Показывает, сколько времени требуется LLM для ответа.

Каждая строка в таблице диалогов — это один вызов LLM, поэтому мы просто строим график по этим значениям:

```sql
SELECT
  timestamp AS time,
  response_time
FROM conversations
WHERE timestamp BETWEEN $__timeFrom() AND $__timeTo()
ORDER BY timestamp
```

Для этой панели используйте тип визуализации "Time series".

## Панель использования токенов (Token Usage)

Показывает потребление токенов во времени.

Если диапазон времени большой, точек будет слишком много для отображения каждой по отдельности.

Мы сгруппируем данные по интервалам (например, каждые 5 минут) и возьмем среднее значение внутри каждого интервала:

```sql
SELECT
  $__timeGroup(timestamp, $__interval) AS time,
  AVG(total_tokens) AS avg_tokens
FROM conversations
WHERE timestamp BETWEEN $__timeFrom() AND $__timeTo()
GROUP BY 1
ORDER BY 1
```

- `$__timeGroup` округляет метки времени до заданных интервалов.
- `GROUP BY 1` группирует по первой колонке (интервалу).
- `AVG` дает среднее количество токенов на интервал.

Для этой панели используйте тип визуализации "Time series".

## Панель стоимости (Cost)

Показывает суммарную стоимость во времени.

Идея та же, но вместо среднего мы используем `SUM` (сумму) для каждого интервала:

```sql
SELECT
  $__timeGroup(timestamp, $__interval) AS time,
  SUM(cost) AS total_cost
FROM conversations
WHERE timestamp BETWEEN $__timeFrom() AND $__timeTo()
  AND cost > 0
GROUP BY 1
ORDER BY 1
```

Для этой панели используйте тип визуализации "Time series".

## Панель используемых моделей (Model Usage)

Показывает, какие модели используются:

```sql
SELECT
  model,
  COUNT(*) as count
FROM conversations
WHERE timestamp BETWEEN $__timeFrom() AND $__timeTo()
GROUP BY model
```

Используйте тип "Bar chart" (столбчатая диаграмма).

## Панель распределения релевантности (Relevance Distribution)

Показывает количество ответов с метками RELEVANT, PARTLY_RELEVANT или NON_RELEVANT по оценке судьи:

```sql
SELECT
  relevance,
  COUNT(*) as count
FROM feedback
WHERE source = 'judge'
  AND timestamp BETWEEN $__timeFrom() AND $__timeTo()
GROUP BY relevance
```

Используйте тип "Pie chart" (круговая диаграмма).

## Панель отзывов пользователей (User Feedback)

Показывает соотношение "пальцев вверх" и "пальцев вниз":

```sql
SELECT
  SUM(CASE WHEN score > 0 THEN 1 ELSE 0 END) as thumbs_up,
  SUM(CASE WHEN score < 0 THEN 1 ELSE 0 END) as thumbs_down
FROM feedback
WHERE source = 'user'
  AND timestamp BETWEEN $__timeFrom() AND $__timeTo()
```

Используйте тип "Gauge" (индикатор) или "Pie chart".

## Панель последних диалогов (Recent Conversations)

Показывает последние 5 диалогов в виде таблицы:

```sql
SELECT
  timestamp AS time,
  question,
  answer,
  response_time,
  cost
FROM conversations
WHERE timestamp BETWEEN $__timeFrom() AND $__timeTo()
ORDER BY timestamp DESC
LIMIT 5
```

Тип панели: "Table" (таблица).

## Настройки дашборда

Установите автообновление дашборда каждые 30 секунд, чтобы данные всегда были актуальными. Также можно установить диапазон по умолчанию "Last 6 hours" (за последние 6 часов).

Расположите панели логично:

- Верхний ряд: таблица последних диалогов (широкая)
- Средний ряд: столбчатая диаграмма моделей | круговая диаграмма релевантности
- Нижний ряд: время ответа | использование токенов | стоимость

Не считайте эти типы панелей единственно верными. Для одного и того же запроса можно попробовать разные визуализации. Экспериментируйте и оставляйте то, что лучше читается. Пока синтетические данные продолжают поступать, вы будете видеть, как графики меняются в реальном времени.

Этот дашборд дает четкое представление о состоянии системы. Скорость ответа, стоимость, релевантность и оценки пользователей — всё в одном месте.
