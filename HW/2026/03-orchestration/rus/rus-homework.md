## Инструкции по настройке

Настройте свои API-ключи как секреты Kestra:

⚠️ Важно: Никогда не фиксируйте API-ключи в Git! Всегда используйте переменные окружения.

Kestra считывает секреты из переменных окружения с префиксом `SECRET_`, где значение закодировано в base64. Экспортируйте свои ключи перед запуском Kestra:

```bash
export GEMINI_API_KEY="ваш-gemini-api-ключ"
export SECRET_GEMINI_API_KEY=$(echo -n "ваш-gemini-api-ключ" | base64)
export SECRET_TAVILY_API_KEY=$(echo -n "ваш-tavily-api-ключ" | base64)  # необязательно
```

Затем запустите (или перезапустите) Kestra:

```bash
cd 03-orchestration
docker compose up -d
```

В потоках секреты указываются как `{{ secret('GEMINI_API_KEY') }}` — опускайте префикс `SECRET_` при вызове функции `secret()`.

Импортируйте потоки для домашнего задания:

```bash
cd 03-orchestration

# Отрегулируйте имя пользователя и пароль в соответствии с вашими настройками Kestra
curl -X POST -u 'admin@kestra.io:Admin1234!' http://localhost:8080/api/v1/flows/import -F fileUpload=@flows/1_chat_without_rag.yaml
curl -X POST -u 'admin@kestra.io:Admin1234!' http://localhost:8080/api/v1/flows/import -F fileUpload=@flows/2_chat_with_rag.yaml
curl -X POST -u 'admin@kestra.io:Admin1234!' http://localhost:8080/api/v1/flows/import -F fileUpload=@flows/4_simple_agent.yaml
curl -X POST -u 'admin@kestra.io:Admin1234!' http://localhost:8080/api/v1/flows/import -F fileUpload=@flows/5_web_research_agent.yaml
curl -X POST -u 'admin@kestra.io:Admin1234!' http://localhost:8080/api/v1/flows/import -F fileUpload=@flows/6_multi_agent_research.yaml
```

## Вопрос 1: Проектирование контекста (Context Engineering)

Проведите следующий эксперимент:

1. Откройте ChatGPT в окне инкогнито: https://chatgpt.com
2. Введите промпт: "Create a Kestra flow that loads NYC taxi data from CSV to BigQuery"
3. Затем используйте AI Copilot в Kestra с тем же промптом.

Какова основная причина того, что AI Copilot генерирует более качественные потоки Kestra по сравнению с ChatGPT после использования одного и того же промпта?

- AI Copilot использует более мощную модель
- AI Copilot имеет доступ к актуальной документации плагинов Kestra
- AI Copilot использует больше токенов
- AI Copilot имеет доступ к интернету

## Вопрос 2: Сравнение RAG

Запустите оба потока `1_chat_without_rag.yaml` и `2_chat_with_rag.yaml` и сравните их результаты. В обоих случаях задается вопрос: "Which features were released in Kestra 1.1?"

Какую разницу вы наблюдаете?

- Версия с RAG предоставляет конкретные, точные детали функций, основанные на документации
- Оба потока выдают идентичные результаты
- Версия без RAG более детализирована и точна
- Версия с RAG галлюцинирует больше функций, чем версия без RAG

## Вопрос 3: Использование токенов

Запустите `4_simple_agent.yaml` дважды:

1. Первый раз с `summary_length` = `short`
2. Второй раз с `summary_length` = `long`

Проверьте использование токенов, зафиксированное в логах в конце каждого запуска. Как различается использование токенов между коротким и длинным резюме для задачи `multilingual_agent`?

- Существенной разницы нет (в пределах 10% отклонения)
- Длинное резюме использует в 2-4 раза больше выходных токенов, чем короткое
- Короткое резюме использует больше токенов из-за сложности сжатия
- Использование токенов идентично независимо от длины

## Вопрос 4: Автономность агента

Запустите `5_web_research_agent.yaml` с темой исследования по умолчанию о трендах в оркестрации данных.

Кто в этом потоке решает, когда использовать инструмент веб-поиска?

- Разработчик рабочего процесса указывает точный порядок использования инструментов в YAML
- Агент решает автономно на основе промпта и системного сообщения
- Инструменты вызываются LLM случайным образом
- Веб-поиск запускается автоматически при каждом выполнении агента

## Вопрос 5: Мультиагентное взаимодействие

Изучите `6_multi_agent_research.yaml` и запустите его для компании по умолчанию (kestra.io).

Какова роль агента-исследователя (research agent) в этой мультиагентной системе?

- Он принимает окончательные решения об анализе компании и структурирует вывод
- Он служит инструментом для основного агента для сбора данных из сети
- Он обобщает выводы основного агента в отчет
- Он проверяет вывод основного агента на точность

## Вопрос 6: Лучшие практики

Основываясь на том, что вы узнали в этом модуле, какой подход наиболее уместен для рабочих процессов в продакшене, требующих детерминированных, повторяемых результатов со строгими требованиями к комплаенсу (например, финансовая отчетность, рабочие процессы в высоко регулируемых отраслях)?

- Всегда использовать ИИ-агентов для максимальной гибкости и адаптивности
- Использовать традиционные рабочие процессы на основе задач для предсказуемости и возможности аудита
- Использовать только RAG без агентов для лучшей производительности
- Использовать исключительно инструменты веб-поиска для обеспечения актуальности данных

## Советы для успеха

1. API-ключи: Убедитесь, что ваш API-ключ Gemini правильно сохранен в KV Store.
2. Лимиты бесплатного уровня: Если вы столкнетесь с ограничениями скорости (rate limits), подождите несколько минут и попробуйте снова.
3. Отладка: Включите `logRequests` и `logResponses` в конфигурации вашего провайдера, чтобы видеть, что отправляется в LLM.
4. Мониторинг затрат: Проверяйте использование токенов в логах выполнения, чтобы понимать расходы.
5. Сообщество: Задавайте вопросы в Slack курса, если застрянете.
