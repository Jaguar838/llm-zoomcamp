# Домашнее задание: dlt

В этом домашнем задании мы возьмем FAQ-агента из Модуля 1,
инструментируем его с помощью [Pydantic Logfire](https://logfire.dev) для
наблюдаемости (observability),
а затем извлечем данные трассировки с помощью dlt и проанализируем их.

В Модуле 1 мы писали цикл агента вручную, а затем познакомились с toy-ai-kit —
агентным фреймворком.

Для этого домашнего задания мы переписали код на [Pydantic AI](https://ai.pydantic.dev/),
чтобы его было проще интегрировать с Logfire. Pydantic AI и Logfire
отлично работают вместе, поэтому мы используем их здесь.

В Модуле 5 мы изучаем мониторинг и наблюдаемость и внедряем
собственное решение для мониторинга. Logfire является альтернативой этому подходу.

Pydantic AI, который не рассматривали ранее.
Концептуально здесь нет ничего нового: мы уже изучили всё необходимое в Модуле 1.
Комментарии в файле объяснят, что происходит и как всё соотносится с тем,
что мы учили ранее.

Обязательно прочитайте код перед продолжением.

## Настройка

Начнем с конфигурации нашего проекта:

```bash
uv init
uv add openai minsearch requests python-dotenv pydantic-ai logfire
uv add "dlt[duckdb]"
```

Откройте `.env` и добавьте ваш `OPENAI_API_KEY`:

```bash
OPENAI_API_KEY=sk-YOUR_KEY_HERE
```

Убедитесь, что файл в `.gitignore`:

```
.env
```

Вместо OpenAI вы можете использовать любого другого провайдера.

Убедитесь, что агент запускается:

```bash
uv run python main.py
```

## Вопрос 1. Инструментирование агента с помощью Logfire

Зарегистрируйте бесплатную учетную запись [Logfire](https://logfire.dev), создайте
проект и сгенерируйте токен записи (write token). Добавьте его в `.env` как
`LOGFIRE_TOKEN`.

Инструментируйте агента:

```python
logfire.configure()
logfire.instrument_pydantic_ai()
```

Запустите агента несколько раз с разными вопросами и откройте свой
проект в Logfire, чтобы увидеть трассировки (traces).

Для следующего запроса

> How do I run Ollama locally?

сколько спанов (spans) генерирует один запуск агента?

Каждый спан — это либо сам запуск агента, либо вызов LLM, либо вызов инструмента.
Количество может варьироваться от запуска к запуску, так как модель сама решает,
сколько раз выполнять поиск.

* 1
* 5
* 15
* 30

## Вопрос 2. Загрузка трассировок в DuckDB с помощью dlt

Сгенерируйте токен чтения (read token) для вашего проекта Logfire и установите его как
`LOGFIRE_READ_TOKEN` в `.env`.

Инициализируйте проект dlt-hub, как на воркшопе. Затем попросите своего ИИ-помощника для кодинга извлечь данные из Pydantic Logfire и сохранить их в DuckDB.

У dltHub AI workbench есть готовый контекст для Logfire. Направьте вашего агента на него: https://dlthub.com/context/source/logfire

Если вы сейчас не используете ИИ-помощника для кодинга, вы можете использовать что-то вроде OpenCode: вы должны быть в состоянии завершить одну сессию с бесплатной учетной записью.

В качестве альтернативы вы можете сделать это старым способом (используя ChatGPT или вашу любимую поисковую систему).

Трассировки Logfire содержат глубоко вложенный JSON (атрибуты спанов с сообщениями LLM, вызовами инструментов, использованием токенов и т. д.). dlt автоматически нормализует это в набор таблиц — одну для основных записей, плюс дочерние таблицы для каждого уровня вложенности.

Сколько таблиц создал dlt? Проверьте с помощью:

```sql
SELECT COUNT(*) FROM information_schema.tables 
WHERE table_schema = 'agent_traces';
```

* 1
* 3
* 24
* 100

## Вопрос 3. Запрос трассировок с помощью агента

Используя ИИ-помощника для кодинга (вы также можете написать код вручную), найдите
количество входных токенов для запуска агента из Вопроса 1.

Количество токенов хранится в атрибутах спана как
`gen_ai.usage.input_tokens`. Суммируйте их по всем вызовам LLM внутри
трассировки. Число зависит от того, сколько поисковых запросов сделал агент,
поэтому укажите диапазон, в который оно попадает:

* 100 - 500
* 1500 - 5000
* 10000 - 20000
* 50000 - 100000
